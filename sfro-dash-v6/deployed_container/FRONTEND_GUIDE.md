# SFRO Dashboard: Frontend Integration Guide (`index.html`)

This document serves as the canonical reference for the `index.html` frontend rendering engine. 

The dashboard is designed to be a "dumb client" that receives structured JSON data and dynamically re-renders itself. You can completely change the layout and purpose of most cards on the screen simply by changing the JSON payload generated by your backend python scripts, without ever editing the HTML.

## 1. Grid Layout & Card Containers

The dashboard is built on a responsive 12x12 CSS Grid layout. 
Each distinct area of the dashboard is an empty shell called a "Card", identified by a unique HTML ID.

### Existing Card Registry Mapping
By default, the `fetchAndRenderAll()` function maps these card IDs to specific JSON files in the `v5-test/data/` (or `data/`) directory:

| Card ID | Default JSON Source | Default Purpose |
|---------|---------------------|-----------------|
| `#skycam-card` | `skycam.json` | Live All-Sky Camera feed |
| `#fra400img-card` | `fra400.json` | FRA400 latest capture image |
| `#q75img-card` | `75q.json` | 75Q latest capture image |
| `#atmos-left-card` | `atmos_left.json` | Weather/Tempest metrics list |
| `#atmos-right-card` | `atmos_right.json` | Secondary metrics list |
| `#fra400cap-card` | `fra400cap.json` | Text log for FRA400 captures |
| `#q75cap-card` | `q75cap.json` | Text log for 75Q captures |
| `#forecast-card` | `forecast.json` | Horizontal weather timeline |
| `#roofbox-card` | `roof.json` | Roof open/close status |

## 2. The JSON Rendering Engine (`renderCard`)

Every 30 seconds, `index.html` fetches the JSON files and passes them to the `renderCard(cardId, data)` Javascript function. 

This function acts as a **Dynamic Component Switcher**. It looks at the `"type"` field inside your JSON payload to determine how to format the card.

### Universal Text Fallback
**Crucially, every card supports the `"text"` type.** 
If a script fails, or if a camera goes offline, or if you simply want to repurpose an Image Box into a text log, you just change the JSON payload type to `"text"`. The javascript engine will automatically hide the image/list/status elements and reveal a text container.

### Card Title Restoration (`data-orig`)
When a card is overridden by custom content (using `custom_<cardname>.json`), its title is replaced by the custom payload's `"title"`. To prevent this title change from being "sticky" when reverting back to standard automated cards:
* The frontend caches the native HTML title on the card block using a `data-orig` attribute during its first render.
* If a JSON payload arrives with an empty/null `"title"` (which is the case when custom overrides are deleted and standard data returns), the frontend automatically restores the native HTML title cached in `data-orig`.

## 3. Supported JSON Payloads

Your JSON files must adhere to this root structure:

```json
{
  "title": "Optional Card Title",
  "subtitle": "Optional Subtitle/Target",
  "type": "image|text|list|forecast|status",
  "data": { 
     // Data payload specific to the type
  }
}
```

### Type: `"image"`
Renders an image. Fills the card.
```json
{
  "title": "FRA400",
  "subtitle": "NGC 6871",
  "type": "image",
  "data": {
    "src": "https://example.com/latest_capture.jpg",
    "alt": "Latest FRA400 Capture"
  }
}
```

### Type: `"text"`
Hides any specialized elements (images, lists) and displays a dynamically resizing text block.
*Supported by: All Cards.*
```json
{
  "title": "FRA400 Status",
  "subtitle": "Offline",
  "type": "text",
  "data": {
    "text": "The camera is currently disconnected.\nPlease check the USB hub."
  }
}
```

### Type: `"list"`
Displays a 2-column key/value list. Primarily used by the Atmospheric parameters cards.
```json
{
  "title": "Atmospheric Parameters",
  "type": "list",
  "data": {
    "items": [
      { "label": "Wind", "value": "7.6 mph", "color": "#fff" },
      { "label": "Humidity", "value": "47%", "color": "#FF9800" }
    ]
  }
}
```

### Type: `"status"`
Displays a large, centered status text (e.g., Roof Status) with optional smaller details text below it.
```json
{
  "title": "Roof Status",
  "type": "status",
  "data": {
    "status": "OPEN",
    "color": "#10b981",
    "details": "Opened at 06:52 Central\nNo rain detected."
  }
}
```

### Type: `"forecast"`
Displays a horizontal weather timeline.
*Note: Currently, the HTML scaffolding for this specific layout is only physically present inside the `#forecast-card`.*
```json
{
  "title": "YR Forecast",
  "subtitle": "Wandsworth",
  "type": "forecast",
  "data": {
    "timeline": [
      {
        "time": "14:00",
        "symbol_code": "clearsky_day",
        "temp": 32,
        "cloud": 10,
        "precip": 0.0,
        "wind": 15
      },
      {
        "time": "20:45",
        "event": "Sunset"
      }
    ]
  }
}
```
